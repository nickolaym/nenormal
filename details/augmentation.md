# Расширение модели вычислений

Изначально НАМ-машина работает со строками.

```
Rule : string -> fail | success{string, state}
Machine : string -> string
```

Дополение в духе монады Writer выглядит следующим образом

```
Rule : Augmented string -> fail | success(Augmented string, state)
Machine : Augmented string -> augmented string
```

Строка оборачивается в структуру `augmented_text` вместе с произвольным аргументом `aux`.

## Как правила работают с обёрткой

Собственно, есть два способа
- вообще не трогать aux
- трогать aux при успешном срабатывании правила

### Элементарное правило

Элементарное правило (`rule`)
- извлекает текст из структуры,
- выполняет подстановку (которая может закончиться неудачей)
- при неудаче возвращает `fail` как обычно
- при успехе собирает новый `augmented_text` с результатом подстановки и результатом обработки `aux`, и всё это вместе оборачивает в `success`

### Скрытое правило

Скрытое правило (`hidden_rule<Rule auto p>`)
- извлекает текст из структуры,
- применяет к нему своё вложенное правило
- в случае успеха собирает новый `augmented_text` - с новым текстом и старым `aux`, и всё это вместе оборачивает в `success`

Почему это названо "скрытым"? И в чём его смысл?

С его помощью мы скрываем работу вложенного правила (это может быть группа `rules`, например) от аугментации. Вложенные правила работают только со строками.

## Что представляет aux

Поскольку `aux` в первую очередь предназначен для трассировки правил,
то договоримся, что это некоторая колбек-функция, принимающая `(input_text, current_rule, output_text)` и возвращающая новый `aux` (или саму себя).

Трассировать неудачи - нет смысла.

## Виды аугментации

Есть несколько типичных видов аугментации:
- тривиальная (как будто её вовсе нет) - тип `empty` ничего не содержит и ничего не делает
- побочные эффекты - `side_effect{f}`, где `f` - функция, возвращающая void (например, вывод в std::cout или запись в связанную переменную)
- кумулятивные эффекты - `cumulative_effect{f, a}`, где `f` - функция, принимающая аргумент `a` и возвращающая его новое значение.

`empty` и `side_effect` возвращают сами себя, а `cumulative_effect` возвращает новый `cumulative_effect` (с обновлённым аргументом).

## Аугментация во время компиляции и во время исполнения

Если aux - чистая constexpr-функциональная структура, то и результат выполнения `machine(augmented)` - также чистое constexpr-значение.

Если нет, - то `machine(augmented)` превращается в честный рантаймовый код, с вызовами колбеков.

Значения входного и выходного текста - в любом случае являются честными константами времени компиляции с зависимыми типами CtStr.

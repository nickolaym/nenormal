Смотрите в коде юниттестов.

## Пример: hello world

[hello_world.cpp](../tests/hello_world.cpp)

Самый простой случай. Заменяем независимые подстроки, пока заменяется.

```cpp
constexpr auto program = rules<
    rule<"hello"_ss, "privet"_ss>{},
    rule<"world"_ss, "mir"_ss>{}
>{};

constexpr auto src = "hello, world!"_cts;
constexpr auto dst = program(src);

std::cout << src.value.value << std::endl; // hello world
std::cout << dst.value.value << std::endl; // privet mir
```

## Пример: правильная скобочная последовательность

[correct_bracket_sequence.cpp](../tests/correct_bracket_sequence.cpp)

Это более сложный пример, здесь важен порядок выполнения правил.
1) сперва удаляем все парные скобки
  - "()" -> ""
  - "[()]" -> "[]" -> "" - по индукции
2) затем у нас остаётся ворох непарных скобок, мы их замазываем одним символом
  - "(" / "[" / ... -> "_"
3) затем сокращаем всю цепочку замазок до одиночного символа замазки
  - "__" -> "_"
  - "___" -> "__" -> "_" - по индукции
4) наконец, у нас либо пустая строка, либо символ замазки.
  - "_" -> "FAILURE"

```cpp
constexpr auto reduce_pairs = rules<  // сокращаем парные скобки
    rule<"()"_ss, ""_ss>{},
    rule<"[]"_ss, ""_ss>{},
    rule<"{}"_ss, ""_ss>{}
>{};

constexpr auto unify_unpaired = rules<  // замазываем непарные
    rule<"("_ss, "_"_ss>{},
    rule<")"_ss, "_"_ss>{},
    rule<"["_ss, "_"_ss>{},
    rule<"]"_ss, "_"_ss>{},
    rule<"{"_ss, "_"_ss>{},
    rule<"}"_ss, "_"_ss>{}
>{};

constexpr auto shrink_unpaired = rule<"__"_ss, "_"_ss>{};

constexpr auto failure_message = rule<"_"_ss, "FAILURE"_ss>{};

constexpr program = rules<
    reduce_pairs,
    // если дошли до этого места, то в тексте не осталось парных скобок
    unify_unpaired,
    // если дошли до этого места, то текст - это серия подчёркиваний, либо пустая строка
    shrink_unpaired,
    // если дошли до этого места, то текст - это одиночное подчёркивание, либо пустая строка
    failure_message
    // одиночное подчёркивание заменяем на слово FAILURE
>{};

static_assert(program(""_cts) == ""_cts);
static_assert(program("()({})[(()[])]"_cts) == ""_cts);
static_assert(program("()({})[(()[])"_cts) == "FAILURE"_cts);
```

## Пример: сложение десятичных чисел

[arithmetics.cpp](../tests/arithmetics/arithmetics.cpp)

Самый навороченный, на данный момент, пример.

Это иллюстрация возможностей нормальных алгорифмов Маркова.

На входе "X+Y=", где X и Y - неотрицательные целые числа.
На выходе - "Z", где Z - неотрицательная целая сумма.

### Алгоритм:

Пусть исходная строка "Xx+Yy=", где x и y - младшие разряды

1) Возьмём младший разряд x и перенесём его рядом к y
2) Найдём (x+y) = z или 1z (то есть, с переносом)
3) Вынесем z направо, и распространим перенос влево по Y, если это нужно

Будем повторять, пока не кончатся X и-или Y.

Один такт сложения разрядов - это преобразование строк
- было "Xx+Yy=R" где R - ранее вынесенные разряды суммы
- стало "X+U=zR" где z = (x+y) mod 10, U = Y+((x+y) div 10)

Математически это означает следующее
```
"XXXXXx           +  YYYYYy           =              RRR" - было
(X*10+x)*10^n     + (Y*10+y)*10^n     +              RRR
 X      *10^(n+1) +  Y      *10^(n+1) + (x+y)*10^n + RRR
 X      *10^(n+1) + (Y+c   )*10^(n+1) +  z   *10^n + RRR
"XXXXX            + uUUUUU            =  z           RRR" - стало
```

### Особенности реализации

Хотя алгоритм у нас линейный, правила могут конкурировать.
Поэтому нужно разместить их в правильном порядке.

Так, первый шаг "взять младший разряд" должен находиться после последнего "распространить перенос".
Только тогда мы пойдём на новый круг алгоритма, когда предыдущий закончится.

Поэтому итоговая программа выглядит вот так
1) add_carry
2) add_digits
3) move_digit_to_the_right
4) plus_nothing
5) take_digit
6) cleanup

Хотя сам цикл сложения устроен так:
```
пока X и Y не пусто
    (5) берём младшую цифру "x+" и заносим направо "+[x]"
    пока [x] не достиг "="
        (3) обмениваем "[x]y" на "y[x]"
    (2) складываем и выносим "y[x]=" в "=z" (или с переносом "^=z)
    пока есть перенос "y^"
        (1) распространяем перенос "y^" влево по Y - "u" или "^u"
    (1) когда перенос достигает старшего разряда, "=^", превращаем его в старшую 1 - "=1"

если Y пусто, "X+=R"
    (4) просто убираем "+=" и выходим

если X пусто, "+Y=R"
    (6) убираем "+" и "=" по отдельности и выходим
```

Если бы мы захотели, чтобы выходная строка была "=Z", а не просто "Z",
то пришлось бы ещё ввести правила переноса избыточных разрядов из X или Y

1) "y=" -> "=y"
2) "x+=" -> "+=x"
3) "+" -> "="

Сами подумайте, где они должны располагаться в коде.

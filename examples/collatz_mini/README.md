# Последовательность Коллаца (мини-версия)

Это пример для статьи [/article/habr-article.md]

Минимальная программа, которая просто берёт число в единеричной системе счисления
и преобразует по правилу N/2 | 3N+1.

## Исходная задача

На входе - строка из N единиц.

На выходе - резолюция `ZERO` (для N=0) или `STOP` (последовательность свелась к N=1).

### Программа иллюстрирует следующие свойства:

1. НАМ-машина умеет выполнять сложные циклы (в том числе вложенные)
2. Можно совершать арифметические операции
3. Порядок правил в НАМ-программе имеет принципиальное значение
4. Этот порядок не совпадает с "наивным" последовательным алгоритмом

### Алгоритм

1. Проверяем условия останова
   1. при N = 0, то есть, текст "", заменяем его на `ZERO` и завершаем
   2. при N = 1, то есть, текст `1`, заменяем на `STOP` и завершаем
2. Выполняем деление N на 2 с остатком: `(L, R) = (N div 2, N mod 2)`
   1. Добавляем слева маркер `[div]`
   2. В цикле заменяем `[div]11` на `1[div]` - так, что всегда `L*2 + R = N`
3. Если остаток равен 0, то
   - убираем маркер и получаем число `N' = L`
   - и начинаем сначала
4. Если остаток равен 1, то выполняем умножение: `N' = N*3 + 1 = L*6 + 4`
   1. Добавляем справа маркер `[mul]` и единички
   2. В цикле заменяем `1[mul]` на `[mul]111111` - так, что всегда `N' = L*6 + 4`
5. Когда маркер доходит до левого края,
   - убираем его и получаем число `N' = N*3 + 1`
   - и начинаем сначала

### Порядок правил

Правила упорядочены так, что продолжения циклов предшествуют их началу.
В противном случае НАМ-машина просто расплодила бы маркеры.

Начало внешнего цикла - это ветвление

1. если N >= 2 - начать деление
2. если N >= 1 - остановиться
3. если N >= 0 - тоже остановиться

(НАМ здесь не может проверить на строгое равенство, поэтому делаем сравнения в нужном порядке).

И опять же, начало внешнего цикла - после его тела.

### НАМ-программа

```
"1[mul]"  ->  "[mul]111111" -- 4.2
"[mul]"   ->  ""            -- 5
"[div]11" ->  "1[div]"      -- 2.2
"[div]1"  ->  "[mul]1111"   -- 4.1
"[div]"   ->  ""            -- 3
"11"      ->  "1[div]"      -- 2.1
"1"       ->. "STOP"        -- 1.2
""        ->. "ZERO"        -- 1.1
```

### Аугментация

Она не относится к алгоритму, а служит для трассировки исполнения.

Вообще, компилятор сразу строит план исполнения для конкретной строки,
а затем мы в рантайме вызываем функцию, которая последовательно вызовет колбек
с нужными аргументами.

На каждом шаге выводим строку таблички: что было, что применили и что стало.

И инкрементируем счётчик шагов.

Подробнее про устройство аугментации - см. [аугментация](/details/augmentation.md)

## Версия с маркерами начала-конца строки

Ключевая особенность: мы требуем, чтобы исходная строка была вида `^111...(N шт.)...1$`

Это позволяет переписать НАМ-программу так, что в каждый момент времени применимо только одно правило.

### НАМ-программа

```
^$          ->. ZERO
^1$         ->. STOP

^11         -> ^[run][*2+]11 -- изолировали маркер начала и запустили цикл деления
[*2+]11     -> 1[*2+]        -- цикл деления (слева направо)

[*2+]$      -> [+]$          -- закончили деление: чётное - запустили цикл сложения
1[+]        -> [+]1          -- цикл сложения (справа налево)
^[run][+]   -> ^             -- закончили сложение и убрали изолятор

[*2+]1$     -> [*6+]1111$    -- закончили деление: нечётное - запустили цикл умножения
1[*6+]      -> [*6+]111111   -- цикл умножения (справа налево)
^[run][*6+] -> ^             -- закончили умножение и убрали изолятор
```

Программа намеренно написана неоптимально, зато наглядно.
